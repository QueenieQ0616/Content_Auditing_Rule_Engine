# Content Auditing Rule Engine

> 一个产域解耦的通用内容审核规则引擎，面向**不良价值观**内容识别。

输入文本内容，经规则引擎判定后输出「放行 / 转人工 / 拦截」三态结果及风险标签，对外提供标准化、可调用的 API。

---

## 项目简介

本项目实现一个**规则可配置、产域解耦**的内容识别规则引擎。引擎核心与具体产域解耦——核心只实现一次，产域以可插拔的策略配置形式挂载到引擎上。当前版本聚焦**不良价值观**一个产域（如拜金炫富、低俗审丑、暴力歧视宣扬等不良导向内容），新增产域只需追加策略配置、无需修改引擎代码。

**核心特性**

- **配置化规则引擎**：规则以 DSL（JSON）描述，非硬编码，支持条件组合（AND/OR）与热更新。
- **三态判定**：输出「放行 / 转人工 / 拦截」，对机器无法确信的灰色地带内容转入人工审核兜底。
- **风险分级打分**：规则按危害等级（L1/L2/L3）分级加权，判定结果可解释、可回溯到命中规则。
- **多档处理尺度**：宽松 / 标准 / 严格三档，通过双阈值控制，尺度与规则等级挂钩，具备业务语义。
- **多维区分**：API 支持区分公司（租户）、业务、模态，便于策略隔离与编排。
- **数据闭环**：识别与人工审核结果异步沉淀，用于回流优化词库与策略。

## 架构概览

```
业务输入
   │
   ▼
[接入策略]  参数校验、按 产域/业务/模态 选定规则集
   │
   ▼
[规则引擎]  遍历规则集 → 各条件经匹配器判断 → 按逻辑组合 → 命中判定
   │
   ▼
[策略编排]  收集命中规则 → 分级加权打分 → 标签合并
   │
   ▼
[输出策略]  按「产域 + 尺度」双阈值 → pass / review / reject
   │
   ├──▶ 业务输出（判定结果 + 标签 + 命中规则）
   │
   └──▶ 异步入队 → 数据沉淀 → 回流优化（旁路闭环）
```


### 2. 启动服务

```bash
# 示例
# python -m api.main
```

### 3. 调用检测接口

**请求**

```http
POST /v1/engine/check
Content-Type: application/json
```

```json
{
  "tenant_id": "company_a",
  "biz_id": "comment",
  "modality": "text",
  "scale": "standard",
  "content": "待检测的文本内容"
}
```

**响应**

```json
{
  "request_id": "req_xxx",
  "decision": "review",
  "risk_level": "medium",
  "score": 0.55,
  "labels": ["低俗审丑"],
  "hit_rules": [
    { "rule_id": "BV_001", "name": "示例规则", "level": "L2", "type": "keyword" }
  ],
  "review_reason": "命中中危规则，分数处于人工审核区间"
}
```

| 字段 | 说明 |
|---|---|
| `decision` | 判定结果：`pass`（放行）/ `review`（转人工）/ `reject`（拦截） |
| `risk_level` | 风险等级 |
| `score` | 风险分数（0~1），由命中规则分级加权得出 |
| `labels` | 风险标签 |
| `hit_rules` | 命中规则明细，可回溯判定依据 |
| `scale`（请求） | 处理尺度：`loose` / `standard` / `strict` |

## 规则配置说明

规则以 JSON 描述，由「条件 + 逻辑 + 动作」构成：

```json
{
  "rule_id": "BV_001",
  "name": "示例规则",
  "domain": "bad_values",
  "level": "L2",
  "weight": 0.6,
  "enabled": true,
  "logic": "AND",
  "conditions": [
    { "type": "keyword", "match": "any", "value": ["示例词A", "示例词B"] },
    { "type": "regex", "value": "正则表达式" }
  ],
  "action": {
    "labels": ["低俗审丑"],
    "score": 0.6
  }
}
```

- **level**：危害等级 L1（低危）/ L2（中危）/ L3（高危）。
- **weight**：权重，用于打分。打分采用「取命中规则最高权重 + 命中数微调」。
- **logic**：`AND`（全部命中）/ `OR`（任一命中）。
- **type**：条件类型，决定由哪种匹配器判断（`keyword` / `regex` / `length` / `model` …），可扩展。

## 处理尺度

三档尺度通过双阈值控制，并与规则等级挂钩，便于解释：

| 尺度 | 行为（示意） |
|---|---|
| 宽松 loose | 仅 L3 高危拦截，L2 转人工，L1 放行 |
| 标准 standard | L3 拦截，L2 转人工，L1 放行 |
| 严格 strict | L2、L3 拦截，L1 转人工，几乎不放行 |

> 不良价值观产域边界模糊，建议默认尺度下保留较大的「转人工」区间，由人工兜底。
