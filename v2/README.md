# 广告识别规则引擎 - C部分

## 项目概述

广告识别规则引擎的 **C部分（接口与前端）** 实现，包含完整的 FastAPI 后端 API 和 Vue3 + Element Plus 前端可视化后台。

引擎对输入内容进行广告识别，判断是否包含广告行为，输出三态结果（放行/转人工/拦截）。

## 技术栈

| 模块 | 技术 |
|------|------|
| 后端 | Python + FastAPI |
| 前端 | Vue3 + Element Plus + Vite |
| 通信 | RESTful API + Axios |

## 快速开始

### 1. 启动后端服务

```bash
cd backend

# 创建虚拟环境（推荐）
python -m venv venv

# Windows
venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 启动服务
python main.py
```

后端服务将在 `http://localhost:8000` 启动，自动 API 文档地址：
- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

### 2. 启动前端服务

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端将在 `http://localhost:3000` 启动，并自动代理 API 请求到后端。

## 核心设计

### 三态结果
- **放行（pass）**：内容正常，非广告
- **转人工（manual）**：疑似广告，需人工复核
- **拦截（block）**：明确广告，直接拦截

### 危害分级
- **L1（低危）**：疑似营销、模糊引流
- **L2（中危）**：软广植入、效果夸大、直播间引流
- **L3（高危）**：硬广推销、联系方式、价格诱导

### 处理尺度
- **宽松**：只拦截最明显的硬广
- **标准**：平衡准确率和召回率（默认）
- **严格**：宁可错杀，多拦截、多转人工

### 广告规则分类

| 分类 | 规则 | 等级 |
|------|------|------|
| 硬广推销 | 联系方式（加微信、VX号等） | L3 |
| 硬广推销 | 价格诱导（限时特惠、亏本甩卖等） | L3 |
| 软广植入 | 产品推荐（强烈推荐、亲测好用等） | L2 |
| 软广植入 | 效果夸大（三天见效、好评如潮等） | L2 |
| 引流营销 | 直播间引流（进直播间、直播福利等） | L2 |
| 疑似广告 | 模糊营销（私信了解、戳链接等） | L1 |
