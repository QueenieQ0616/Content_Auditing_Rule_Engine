# 广告识别规则引擎 - 数据与策略工作文档

**角色**：B（数据与策略）  
**项目目标**：构建不良价值观识别引擎，聚焦广告识别方向  
**最终交付**：规则配置、测试集、评估脚本及调优报告

本文档详细记录了从词库收集、等级分类、测试集构建到评估调优的全过程，包括技术选型、具体操作和迭代改进。

## 一、词库合并与预处理

### 1.1 词库来源
从两个开源仓库获取原始词库：
- **Sensitive-lexicon**：通用广告/违禁词库，约 123 词  
- **campus-ad-detection-words**：校园广告/兼职刷单专项词库，约 541 词

合并后得到 **664 个词**，去重后 **606 个独立词**。

### 1.2 词库格式
原始文件为纯文本，每行一个词（UTF-8 编码）。部分词示例：
```
加微信
刷单兼职
裸聊
免费领取
限时抢购
新品上市
```

## 二、危害等级分类（L1/L2/L3）

### 2.1 分类策略选择
首次尝试使用 **Sentence-BERT 语义模型**（`shibing624/text2vec-base-chinese`）进行自动分类。  
- 预定义每个等级的**种子词短语**（L3 含色情、赌博、硬广；L2 含营销诱导；L1 含中性促销）。  
- 计算输入词与种子词的余弦相似度，取最高分类。  
- 设定相似度阈值 `CONFIDENCE_THRESHOLD = 0.35`（低于阈值归为 L1）。

**初始分类结果**：
```
L3=21, L2=136, L1=449
```
问题：大量色情词（如“裸聊”）仍被分到 L1，原因是模型对短词语义捕捉不稳。

### 2.2 加入强制规则修正
在语义分类前增加**正则强制规则**，优先将明确的高危词归入 L3：

```python
FORCE_RULES = [
    (re.compile(r"(裸聊|约炮|一夜情|上门服务|色情网站|AV|无码|成人用品|...)"), "L3"),
    (re.compile(r"(加微信|加QQ|刷单|日结|博彩|六合彩)"), "L3"),
    (re.compile(r"(免费领取|限时抢购|爆款|种草|网红同款)"), "L2"),
]
```

调整后 L3 数量提升至 **60+**，主要色情词被正确捕获。

### 2.3 最终等级分布（基于规则分类）
考虑到语义模型的不稳定性，**后期完全切换到基于关键词的正则分类**，采用以下规则：

| 等级 | 特征关键词（部分） | 权重 |
|------|-------------------|------|
| L3 | 加微信、裸聊、刷单、贷款、代写、代考、赌博、色情、手机号 | 0.9 |
| L2 | 免费领取、限时抢购、种草、网红同款、零风险 | 0.6 |
| L1 | 新品上市、品质保证、包邮、满减、好评返现 | 0.3 |

最终生成 `rules/advertising.json`，每条规则对应一个等级，使用 `OR` 逻辑匹配关键词列表。

## 三、测试集构建

### 3.1 数据来源
- **正样本**：从以下渠道收集广告文本（约 120 条）  
  - QQ 群公告、校园兼职信息、贷款广告、色情引流等真实样本  
  - 从词库中随机组合生成构造文本（如“加微信看裸聊”）  
- **负样本**：正常校园讨论、学习交流、日常对话（约 100 条）  
- **边界样本**：软广、疑似诱导内容（约 30 条）

### 3.2 标注规范
- `reject`：明确包含联系方式、色情、赌博、刷单、诈骗等严重违规内容  
- `review`：营销语气但无直接联系方式，或疑似软广  
- `pass`：正常内容，无广告意图  

最终测试集规模：**250 条**（实际使用 210 条，由于部分样本格式问题剔除）。

### 3.3 测试集文件格式（`tests/test_set.csv`）
```csv
text,expected
"加微信 abc123 看详情",reject
"今天天气真好",pass
"限时特价，手慢无",reject
"想赚钱的私我",review
...
```

## 四、评估脚本与指标

### 4.1 评估脚本设计（`tests/evaluate.py`）
- 加载引擎（固定使用 `rules/advertising.json`）  
- 遍历测试集，调用引擎的 `check(content, scale)` 方法  
- 计算预测结果与期望的混淆矩阵  
- 输出精准率、召回率、F1 以及前 10 个错误样本

关键代码片段：
```python
class FixedEngine:
    def __init__(self, rules_json_path):
        # 加载指定 JSON 规则，不扫描目录
    def check(self, content, scale="standard"):
        # 匹配规则、打分、三态判定
```

### 4.2 初始评估结果（standard 尺度）
```
已加载 3 条规则（来自 rules/advertising.json）
尺度: standard
拦截精准率: 0.86, 召回率: 0.54, F1: 0.66
TP=30, FP=5, FN=26, TN=101

错误样本数: 40
```
**主要问题**：召回率过低（0.54），大量广告被漏过（FN=26）。错误样本分析发现：
- 联系方式（微信号、QQ号、群号、手机号）未被正则覆盖
- 部分广告词（贷款、代考、刷单、红包等）不在词库中
- 缺乏组合条件（如“免费”+“加微信”）

## 五、策略调整与优化

### 5.1 新增正则规则（解决联系方式漏报）
在 `rules/advertising.json` 中添加三个 L3 正则规则：

| 规则 ID | 正则表达式 | 作用 |
|---------|------------|------|
| `AD_REGEX_WECHAT` | `微信[号]?[:：]?[a-zA-Z0-9_]+` 等 | 匹配微信号、加微信变体 |
| `AD_REGEX_QQ` | `QQ[:：]?[0-9]{5,11}` 等 | 匹配QQ号、群号 |
| `AD_REGEX_PHONE` | `1[3-9]\d{9}` | 匹配手机号 |

### 5.2 扩充关键词库（约30个）
将以下词加入 L3 规则：
```
贷款、代写、代考、代体检、代上课、替考、买文凭、刷单、兼职、日结、返现、红包、注册送、邀请码、点击领取、扫码进群、免费领、限时折扣、爆款推荐、网红爆款、跳楼价、亏本清仓、老板跑路、全场一折、免月租、办理校园卡、分期付款、0首付、租学生证、卖答案
```

### 5.3 增加组合规则（AND 逻辑）
为了识别需要多个特征同时出现的广告（如“免费领取+加微信”），添加以下组合规则：

```json
{
  "rule_id": "AD_COMBO_FREE_ACTION",
  "level": "L3",
  "weight": 0.9,
  "logic": "AND",
  "conditions": [
    { "type": "keyword", "value": ["免费", "限时", "优惠", "领"] },
    { "type": "keyword", "value": ["加", "联系", "扫码", "点击", "私"] }
  ]
}
```

### 5.4 调整权重与阈值
- 将 L2 规则权重从 0.6 提升至 **0.75**，L1 从 0.3 提升至 **0.45**  
- 建议将 `standard` 尺度的 `high` 阈值从 0.7 降至 **0.6**（可选，根据后续评估决定）

## 八、反思与改进方向

1. **语义模型局限性**：短词分类不适合 Sentence-BERT，改用正则+规则后效果更可控。  
2. **组合规则的必要性**：单纯的关键词匹配容易误报，AND 逻辑和正则距离约束能显著提升精度。  
3. **测试集质量**：初期测试集偏小且部分标注不够准确，后续应扩大规模并交叉验证。  
4. **实时反馈循环**：建议将评估中发现的错误样本自动加入词库或规则，形成闭环优化。
